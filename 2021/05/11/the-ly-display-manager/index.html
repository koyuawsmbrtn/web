The ly display manager

11.05.2021

Some time ago I found a nice TTY-based display manager. I know that some folks have put startx into their .bashrc or start it manually which is okay, but not nearly as good as having a full-blown display manager that starts fast and loads everything including your user session. ly is also included in koyuOS, my own operating system based on Arch Linux, because koyuOS should be as leightweight, but easy-to-use as possible. I think I found the middle-ground there. I think choosing ly was a better decision than lightdm, because display managers like these already load up a whole X server in itself which makes the bootup process a little slower. My current koyuOS installation is using EFISTUB so I don’t have to use bootloaders and all that crap to load up my OS, the EFI firmware on-board handles it all nicely for me. I also disabled NetworkManager-wait-online.service, because it heavily influenced boot times by a few seconds. I hope I could help and you should definitely use ly, it’s a great display manager.
